# 01. 변수, 데이터 타입, 연산자

JavaScript 코드의 가장 기본적인 구성 요소는 값을 저장하는 **변수(Variable)**, 값의 종류를 나타내는 **데이터 타입(Data Type)**, 그리고 이 값들을 조작하는 **연산자(Operator)**입니다.

#학습목표

- `let`, `const`, `var`를 사용하여 변수와 상수를 선언하고 차이점을 설명할 수 있습니다.
- JavaScript의 주요 데이터 타입(Number, String, Boolean, Null, Undefined, Object)을 이해하고 `typeof` 연산자로 타입을 확인할 수 있습니다.
- 동등 연산자(`==`)와 일치 연산자(`===`)의 차이점을 이해하고 올바르게 사용할 수 있습니다.
- Truthy와 Falsy의 개념을 이해하고 조건문에서 어떻게 동작하는지 설명할 수 있습니다.

---

## 1. 변수와 상수 선언

#변수 #variable #상수 #constant

데이터를 저장하고 참조하기 위해 사용되는 식별자입니다. ES6(2015) 이후로는 `let`과 `const`를 사용하는 것이 표준입니다.

| 키워드  | 특징             | 스코프(유효 범위) | 재할당 | 재선언 | 호이스팅 |
| :------ | :--------------- | :---------------- | :----- | :----- | :------- |
| `let`   | 변수 선언        | 블록 `{}`         | 가능   | 불가능 | X        |
| `const` | 상수 선언        | 블록 `{}`         | 불가능 | 불가능 | X        |
| `var`   | (구식) 변수 선언 | 함수              | 가능   | 가능   | O        |

- **`let`**: 재할당이 가능한 변수를 선언합니다. 값이 변경될 수 있는 경우 사용합니다.
  ```javascript
  let count = 0;
  count = 1; // 가능
  // let count = 2; // 불가능 (SyntaxError: Identifier 'count' has already been declared)
  ```
- **`const`**: 재할당이 불가능한 상수(constant)를 선언합니다. 한 번 할당된 값은 변경할 수 없습니다. 프로그램의 안정성을 위해 기본적으로 `const`를 사용하고, 재할당이 필요한 경우에만 `let`을 사용하는 것이 좋습니다.

  ```javascript
  const PI = 3.14;
  // PI = 3.14159; // 불가능 (TypeError: Assignment to constant variable.)

  const user = { name: "Kim" };
  user.name = "Lee"; // 객체의 속성을 변경하는 것은 가능
  // user = { name: 'Park' }; // 객체 자체를 재할당하는 것은 불가능
  ```

- **`var`**: 오래된 방식의 변수 선언 키워드입니다. 함수 스코프를 가지며, 변수 선언이 코드의 최상단으로 끌어올려지는 **호이스팅(Hoisting)** 문제가 있어 예기치 않은 동작을 유발할 수 있습니다. **현대 JavaScript 개발에서는 사용을 지양합니다.**

## 2. 데이터 타입 (Data Types)

#데이터타입 #data_type

JavaScript는 **동적 타입(Dynamically Typed)** 언어로, 변수에 할당되는 값에 따라 타입이 결정됩니다.

### 원시 타입 (Primitive Types)

- **`Number`**: 정수와 실수를 모두 포함하는 숫자 타입. (`Infinity`, `-Infinity`, `NaN` 포함)
- **`String`**: 텍스트 데이터를 나타내는 문자열 타입. 작은따옴표(`'`)나 큰따옴표(`"`)로 감쌉니다.
- **`Boolean`**: `true` 또는 `false` 값을 가지는 논리 타입.
- **`Null`**: '값이 없음'을 의도적으로 명시하는 타입. `null` 자체가 하나의 값입니다.
- **`Undefined`**: 변수를 선언하고 값을 할당하지 않았을 때, 또는 객체에 존재하지 않는 속성에 접근할 때의 기본값.
- **`Symbol`**: ES6에서 추가된, 유일하고 변경 불가능한 원시 값. 주로 객체 속성의 충돌을 방지하기 위해 사용됩니다.
- **`BigInt`**: `Number`가 표현할 수 있는 한계를 넘어서는 매우 큰 정수를 다루기 위한 타입.

### 객체 타입 (Object Type)

- **`Object`**: 여러 값이나 복잡한 개체를 표현할 수 있는 타입. `{ key: value }` 형태의 속성(property)들로 구성됩니다. 배열, 함수, 정규표현식 등도 모두 객체입니다.

### `typeof` 연산자

변수의 데이터 타입을 확인하는 데 사용됩니다.

```javascript
typeof 42; // "number"
typeof "hello"; // "string"
typeof true; // "boolean"
typeof undefined; // "undefined"
typeof { a: 1 }; // "object"
typeof [1, 2, 3]; // "object" (배열도 객체의 한 종류)
typeof function () {}; // "function" (함수는 특별한 종류의 객체)
typeof null; // "object" (JavaScript 초기 버전의 유명한 버그)
```

## 3. 주요 연산자 (Operators)

### 산술 연산자: `+`, `-`, `*`, `/`, `%`, `**` (거듭제곱)

### 할당 연산자: `=`, `+=`, `-=`, `*=`, `/=`, `%=`

### 비교 연산자

- **`==` (동등 연산자)**: 두 피연산자의 **값**이 같은지 비교합니다. 비교 과정에서 **암시적 타입 변환**이 일어날 수 있어 예기치 않은 결과를 낳을 수 있습니다.
- **`===` (일치 연산자)**: 두 피연산자의 **값과 타입이 모두** 같은지 비교합니다. 타입 변환을 일으키지 않으므로 더 엄격하고 예측 가능합니다. **특별한 경우가 아니면 항상 `===`를 사용하는 것이 좋습니다.**

```javascript
1 == "1"; // true (문자열 "1"이 숫자 1로 변환되어 비교)
1 === "1"; // false (타입이 다르므로 false)

0 == false; // true
0 === false; // false

null == undefined; // true (특별 규칙)
null === undefined; // false
```

### 논리 연산자: `&&` (AND), `||` (OR), `!` (NOT)

### Truthy & Falsy

#truthy #falsy
JavaScript에서는 `Boolean` 값이 아니더라도 조건문에서 `true` 또는 `false`로 평가되는 값들이 있습니다.

- **Falsy 값**: `false`, `0`, `-0`, `0n`(BigInt), `""`(빈 문자열), `null`, `undefined`, `NaN`
- **Truthy 값**: Falsy가 아닌 모든 값. (e.g., `"hello"`, `1`, `[]`, `{}`)

이 특성은 `||` 연산자와 결합하여 기본값을 설정하는 데 유용하게 사용됩니다.

```javascript
const name = receivedName || "Guest"; // receivedName이 falsy(null, undefined 등)하면 "Guest"가 할당됨
```
