# 00. HTML 개요 및 기본 구조

웹 페이지의 뼈대를 이루는 HTML의 기본적인 개념과 문서 구조에 대해 알아봅니다. 모든 웹 개발의 시작점인 HTML의 역할과 핵심 문법을 이해하는 것을 목표로 합니다.

#학습목표

- HTML이 무엇인지 정의하고 웹 개발에서의 역할을 설명할 수 있습니다.
- 태그(Tag), 요소(Element), 속성(Attribute)의 차이를 이해하고 올바르게 사용할 수 있습니다.
- 기본적인 HTML 문서의 구조(`<!DOCTYPE>`, `<html>`, `<head>`, `<body>`)를 작성할 수 있습니다.
- 메타데이터의 중요성을 이해하고 `charset`, `viewport` 등 필수 메타 태그를 설정할 수 있습니다.

---

## 1. HTML이란?

#HTML #HyperText_Markup_Language #에이치티엠엘

**HTML(HyperText Markup Language)**은 웹 브라우저가 웹 페이지의 콘텐츠를 이해하고 렌더링할 수 있도록 구조와 의미를 정의하는 **마크업 언어**입니다. 프로그래밍 언어처럼 로직을 처리하는 것이 아니라, "이것은 제목이다", "이것은 단락이다" 와 같이 콘텐츠의 역할을 선언합니다.

- **HyperText**: 문서 간에 이동할 수 있는 링크(하이퍼링크)를 의미합니다.
- **Markup Language**: 태그(Tag)를 사용하여 문서의 구조와 서식을 명시하는 언어입니다.

HTML은 웹 페이지의 **뼈대**를 구성하며, **CSS(Cascading Style Sheets)**가 디자인(스타일)을, **JavaScript**가 동적인 기능(동작)을 담당하여 함께 현대적인 웹 프론트엔드를 완성합니다.

## 2. 태그(Tag), 요소(Element), 속성(Attribute)

HTML 문서는 이러한 기본 구성 요소들의 집합으로 이루어집니다.

#태그 #tag #요소 #element #속성 #attribute

| 개념                | 설명                                                                                       | 구문 예시                           |
| ------------------- | ------------------------------------------------------------------------------------------ | ----------------------------------- |
| **태그(Tag)**       | 꺾쇠괄호(`< >`)로 둘러싸인 명령어. 콘텐츠의 시작과 끝을 표시하며 의미와 구조를 전달합니다. | `<p>`, `</p>`, `<img />`            |
| **요소(Element)**   | 시작 태그, 콘텐츠, 종료 태그로 구성된 전체. 문서의 한 부분을 완전하게 정의합니다.          | `<p>Hello, world!</p>`              |
| **속성(Attribute)** | 태그에 추가적인 정보나 기능을 부여하는 `이름="값"` 형태의 쌍. 시작 태그 내에 위치합니다.   | `<a href="https://www.google.com">` |

### 간단 예시

```html
<!-- a: 태그 이름, href: 속성 이름, "..." : 속성 값 -->
<a href="https://www.google.com">구글로 이동</a>
```

- 위 코드에서 `<a ...>`는 **시작 태그**, `</a>`는 **종료 태그**입니다.
- `href`는 `a` 태그의 **속성**이며, 링크의 목적지 주소를 지정합니다.
- `<a href="...">구글로 이동</a>` 전체가 하나의 **요소(Element)**를 이룹니다.

## 3. HTML 문서의 기본 구조

모든 HTML 문서는 정해진 뼈대를 가집니다. 이 구조는 브라우저가 문서를 올바르게 해석하기 위한 최소한의 요구사항입니다.

#문서구조 #boilerplate #보일러플레이트

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <!-- 문서의 메타데이터를 담는 영역 -->
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>문서 제목</title>
  </head>
  <body>
    <!-- 실제 브라우저 화면에 표시될 콘텐츠 영역 -->
    <h1>안녕하세요!</h1>
    <p>이곳에 웹 페이지의 내용이 들어갑니다.</p>
  </body>
</html>
```

- **`<!DOCTYPE html>`**: 이 문서가 HTML5 표준에 따라 작성되었음을 선언합니다. 문서의 가장 첫 줄에 위치해야 합니다.
- **`<html>`**: HTML 문서의 최상위 루트 요소입니다. 모든 다른 요소는 `<html>` 태그 내부에 포함됩니다.
  - `lang="ko"` 속성은 문서의 주 사용 언어가 한국어임을 명시하여 검색 엔진 및 스크린 리더에 도움을 줍니다.
- **`<head>`**: 문서 자체에 대한 정보(메타데이터)를 담는 영역입니다. 브라우저 화면에는 직접 표시되지 않습니다.
  - 문자 인코딩, 뷰포트 설정, 문서 제목, 외부 리소스(CSS, Favicon) 연결 등이 포함됩니다.
- **`<body>`**: 실제 브라우저 화면에 표시될 모든 콘텐츠(텍스트, 이미지, 비디오, 링크 등)를 담는 영역입니다.

## 4. `<head>`와 메타데이터

`<head>` 영역은 검색 엔진 최적화(SEO)와 사용자 경험에 중요한 역할을 하는 메타 태그들을 포함합니다.

#메타데이터 #metadata #SEO #검색엔진최적화

- **문자 인코딩 설정**:

  ```html
  <meta charset="UTF-8" />
  ```

  - 문서의 문자 인코딩 방식을 UTF-8로 지정합니다. 한글이나 다른 다국어 문자가 깨지지 않도록 반드시 포함해야 합니다.

- **뷰포트(Viewport) 설정**:

  ```html
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  ```

  - 모바일 기기 등 다양한 화면 크기에서 페이지가 올바르게 보이도록 설정합니다. `width=device-width`는 기기의 화면 너비에 맞추고, `initial-scale=1.0`은 초기 확대 비율을 1로 설정하여 반응형 웹 디자인의 필수 요소로 사용됩니다.

- **문서 제목**:

  ```html
  <title>나의 첫 웹 페이지</title>
  ```

  - 브라우저 탭이나 즐겨찾기 시 표시되는 제목입니다. 검색 결과에서도 중요한 역할을 합니다.

- **외부 리소스 연결**:

  ```html
  <!-- CSS 파일 연결 -->
  <link rel="stylesheet" href="style.css" />

  <!-- 파비콘(Favicon) 연결 -->
  <link rel="icon" href="favicon.ico" />
  ```

  - `<link>` 태그를 사용하여 외부 CSS 파일을 가져오거나, 브라우저 탭에 표시될 작은 아이콘(파비콘)을 연결할 수 있습니다.

다음 문서에서는 `<body>` 태그 내부에 들어가는 다양한 텍스트 및 링크 관련 태그들에 대해 자세히 알아보겠습니다.
