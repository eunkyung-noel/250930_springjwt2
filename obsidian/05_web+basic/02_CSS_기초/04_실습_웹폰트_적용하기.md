# 04. 실습: 웹폰트 적용하기

디자이너가 의도한 타이포그래피를 모든 사용자에게 동일하게 보여주기 위해서는, 사용자의 컴퓨터에 설치되지 않은 글꼴도 웹 페이지에 표시할 수 있어야 합니다. **웹폰트(Web Font)**는 이러한 문제를 해결하기 위해 등장한 기술로, `@font-face` 규칙을 통해 서버에 있는 폰트 파일을 다운로드하여 사용합니다.

#학습목표

- `@font-face` 규칙을 사용하여 외부 폰트 파일을 CSS에 선언할 수 있습니다.
- CDN(Content Delivery Network)의 개념을 이해하고, Google Fonts나 눈누와 같은 서비스를 통해 웹폰트를 쉽게 적용할 수 있습니다.
- `font-display` 속성을 사용하여 폰트 로딩 시의 사용자 경험을 개선할 수 있습니다.

---

## 1. `@font-face` 규칙

#웹폰트 #webfont #폰트페이스 #font_face

`@font-face`는 CSS에 사용자 정의 폰트를 추가하는 규칙입니다. 폰트의 이름(family)을 지정하고, 폰트 파일의 위치(URL)를 알려주는 역할을 합니다.

### 기본 구문

```css
@font-face {
  font-family: "MyCustomFont"; /* 폰트의 이름을 지정 */
  src: url("path/to/myfont.woff2") format("woff2"), /* 최신 브라우저용 */
      url("path/to/myfont.woff") format("woff"); /* 구형 브라우저용 */
  font-weight: normal;
  font-style: normal;
  font-display: swap; /* 폰트 로딩 전략 */
}

/* 선언한 폰트 사용 */
body {
  font-family: "MyCustomFont", sans-serif;
}
```

- **`font-family`**: CSS에서 이 폰트를 호출할 때 사용할 이름을 지정합니다.
- **`src`**: 폰트 파일의 위치(URL)와 포맷을 지정합니다. `woff2`는 압축률이 높아 최신 브라우저에서 우선적으로 사용되며, 하위 호환성을 위해 `woff`를 함께 제공하는 것이 좋습니다.
- **`font-weight` / `font-style`**: 해당 폰트 파일이 어떤 굵기와 스타일을 위한 것인지 명시합니다.
- **`font-display: swap;`**: 폰트가 로딩되는 동안 우선 시스템의 기본 글꼴로 텍스트를 보여주고, 폰트 로딩이 완료되면 웹폰트로 교체합니다. 텍스트가 보이지 않는 현상(FOIT, Flash of Invisible Text)을 방지하여 사용자 경험을 개선하는 중요한 속성입니다.

## 2. 웹폰트 서비스와 CDN 활용하기

직접 폰트 파일을 서버에 올려서 사용하는 것은 번거롭고 트래픽 부담이 있을 수 있습니다. 대부분의 경우, **Google Fonts**나 **눈누(noonnu)**와 같은 웹폰트 서비스가 제공하는 **CDN(Content Delivery Network)**을 활용하는 것이 훨씬 효율적입니다.

#CDN #Content_Delivery_Network

**CDN**은 전 세계 여러 곳에 위치한 서버에 콘텐츠(여기서는 폰트 파일)를 복사해 두고, 사용자와 가장 가까운 서버에서 콘텐츠를 전송하여 로딩 속도를 높이는 기술입니다.

### 실습: '눈누'를 통해 Pretendard 폰트 적용하기

한국어 웹 환경에서 가장 널리 쓰이는 'Pretendard' 폰트를 CDN을 통해 적용해 보겠습니다.

**1. 눈누 사이트에서 코드 복사**

- [눈누의 Pretendard 페이지](https://noonnu.cc/font/Pretendard)에 접속합니다.
- 오른쪽의 '웹폰트로 사용' 부분에 있는 `@font-face` 코드를 복사합니다.

```css
/* 눈누에서 제공하는 Pretendard 폰트 CDN 코드 */
@font-face {
  font-family: "Pretendard-Regular";
  src: url("https://fastly.jsdelivr.net/gh/Project-Noonnu/noonfonts_2107@1.1/Pretendard-Regular.woff")
    format("woff");
  font-weight: 400;
  font-style: normal;
}
```

_참고: 실제 눈누 사이트에서는 더 다양한 굵기의 폰트 코드를 제공합니다._

**2. CSS 파일에 붙여넣고 적용하기**

복사한 `@font-face` 코드를 CSS 파일의 최상단에 붙여넣고, `body`나 원하는 요소의 `font-family` 속성에 해당 폰트 이름을 추가합니다.

**`style.css`**

```css
/* 1. 눈누에서 복사한 @font-face 코드 붙여넣기 */
@font-face {
  font-family: "Pretendard-Regular";
  src: url("https://fastly.jsdelivr.net/gh/Project-Noonnu/noonfonts_2107@1.1/Pretendard-Regular.woff")
    format("woff");
  font-weight: 400;
  font-style: normal;
}

/* 2. body 태그에 웹폰트 적용 */
body {
  font-family: "Pretendard-Regular", sans-serif;
  line-height: 1.6;
}

h1,
h2 {
  /* 제목은 다른 굵기의 Pretendard 폰트를 사용하고 싶다면,
       해당 굵기의 @font-face도 추가로 선언해야 합니다. */
  font-weight: 700; /* 폰트가 지원하는 굵기 값 */
}

/* 폼 요소는 폰트가 상속되지 않는 경우가 많으므로 별도 지정 */
input,
textarea,
button {
  font-family: "Pretendard-Regular", sans-serif;
}
```

**`index.html`**

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>웹폰트 실습</title>
    <link rel="stylesheet" href="style.css" />
  </head>
  <body>
    <h1>안녕하세요, Pretendard 폰트입니다.</h1>
    <p>
      이 문서는 눈누의 CDN을 통해 제공되는 Pretendard 웹폰트를 사용하고
      있습니다. 이제 모든 사용자가 동일한 글꼴로 이 페이지를 볼 수 있습니다.
    </p>
    <input type="text" placeholder="입력 필드도 동일한 폰트 적용" />
  </body>
</html>
```

이제 이 HTML 파일을 브라우저에서 열면, 사용자의 컴퓨터에 Pretendard 폰트가 설치되어 있지 않아도 CDN을 통해 폰트 파일을 내려받아 지정된 스타일로 텍스트를 렌더링하게 됩니다.

이것으로 CSS 기초 학습을 마칩니다. 이 지식들을 바탕으로 다양한 레이아웃 기법(Flexbox, Grid)과 애니메이션 등을 학습하면 더욱 다채롭고 전문적인 웹 페이지를 제작할 수 있습니다.
